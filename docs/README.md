# docs

이 폴더는 `boan-sensei`의 설계 메모, 프로젝트 철학, 향후 기능 문서를 정리하기 위한 공간입니다.

> English: This directory is for design notes, project philosophy, and future documentation for `boan-sensei`.

## 철학

`boan-sensei`는 “점검 후보 정리 도구”입니다. 개발자가 놓치기 쉬운 프론트엔드 보안 검토 지점을 빠르게 모아주되, 최종 판단은 사람이 하도록 돕는 것을 목표로 합니다.

## 문서 원칙

- 과장된 보안 판정을 피합니다.
- CLI 출력은 짧게 유지합니다.
- 긴 설명은 Markdown 보고서와 문서에 둡니다.
- AI 코딩툴이 읽어도 컨텍스트를 과하게 쓰지 않도록 구조화합니다.
- 사용자가 직접 확인해야 하는 지점을 분명히 표시합니다.

## 향후 문서 후보

- rule 설계 기준
- risk 분류 기준
- Markdown 보고서 템플릿 설계
- AI 코딩툴 어댑터 사용 예시
- npm 배포 절차
